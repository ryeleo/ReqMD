Usage: rqmd [OPTIONS] [TARGET]...

  Requirements/requirements summary updater and fast interactive status
  editor.

  This tool keeps per-file status summaries synchronized and provides an
  optimized, keyboard-driven workflow for updating requirement statuses across
  a markdown catalog.

  Core behavior: - Scans all markdown files under a requirements directory. -
  Computes status counts from "- **Status:** ..." lines. - Inserts or updates
  the summary block between:     <!-- acceptance-status-summary:start -->
  <!-- acceptance-status-summary:end --> - Supports both batch/check mode and
  interactive status editing.

  Interactive workflow highlights: - Select file, then requirement, then
  status. - Fast single-key input via click.getchar() for menu navigation. -
  Paging keys:     n = next page, p = previous page, u = up, q = quit. -
  Requirement-level next/prev shortcuts at status menu:     n = next
  requirement, p = previous requirement (history-aware). - Optional sort
  toggles (s) at file and requirement selection levels. - Optional
  blocked/deprecated reason prompts when setting those statuses.

  Status model: - 💡 Proposed - 🔧 Implemented - ✅ Verified - ⛔ Blocked - 🗑️
  Deprecated

  Non-interactive usage: - Update a single requirement by id/status
  (optionally scoped by file). - Update multiple requirements in one command
  via repeated --update ID=STATUS. - Useful for automation and agent-driven
  workflows.

  Examples: - Check only (no writes):     rqmd --verify-summaries -
  Interactive mode with emoji headers:     rqmd --emoji-headers - Non-
  interactive single update:     rqmd             --update-id R-TELEMETRY-
  LOG-001             --update-status implemented - Non-interactive bulk
  update:     rqmd             --update R-STEELTARGET-AUDIO-004=implemented
  --update R-STEELTARGET-AUDIO-005=verified - Non-interactive batch update
  from file:     rqmd             --update-file tmp/ac-updates.jsonl

  Notes: - This script expects markdown requirement sections to use "###
  <PREFIX>-..."     headers and "- **Status:** ..." lines. - Header prefixes
  are configurable with --id-namespace and default to AC and R. - If
  click/tabulate are missing, install them with pip3.

Options:
  --verify-summaries              Check whether summaries are up to date
                                  without writing changes.
  -v, --detailed                  Show verbose output with full words.
  --emoji-headers                 Use emoji column headers in terse table
                                  output (may misalign in some terminals).
  --walk / --no-walk              Open interactive file/requirement/status
                                  flow after summary table.
  -u, --filesystem-order          Deprecated compatibility alias; filesystem
                                  ordering is already the default for Select
                                  file.
  --update-status TEXT            Non-interactive mode: target status (label,
                                  plain text, or slug, e.g. 'Implemented' or
                                  'verified').
  --update-id TEXT                Non-interactive mode: requirement id to
                                  update, for example R-TELEMETRY-LOG-001.
  --update TEXT                   Non-interactive bulk mode: repeat ID=STATUS
                                  (for example --update
                                  R-FOO-001=implemented).
  --dry-run                       Preview mutation changes without writing
                                  files (applies to --update/--update-
                                  file/--update-priority/--update-
                                  flagged/--seed-priorities).
  --update-file TEXT              Non-interactive batch mode: path to
                                  .jsonl/.csv/.tsv with rows containing
                                  requirement_id/requirement_id/id/req_id/r_id
                                  and status.
  --scope-file TEXT               Optional file scope for non-interactive
                                  updates, repo-relative path such as
                                  docs/requirements/telemetry.md.
  --blocked-note TEXT             Optional reason text when setting status to
                                  Blocked in non-interactive mode.
  --deprecated-note TEXT          Optional reason text when setting status to
                                  Deprecated in non-interactive mode.
  --update-priority TEXT          Non-interactive priority mode: repeat
                                  ID=PRIORITY (for example --update-priority
                                  R-FOO-001=p0).
  --update-flagged TEXT           Non-interactive flagged mode: repeat
                                  ID=true|false (for example --update-flagged
                                  R-FOO-001=true).
  --focus-priority                Interactive mode: default to Priority focus
                                  instead of Status focus in entry panels.
  --priority-rollup               Include priority-aware aggregates (counts by
                                  priority level) in summary blocks.
  --status TEXT                   Filter by status (repeatable/comma-
                                  separated).
  --priority TEXT                 Filter by priority (repeatable/comma-
                                  separated).
  --flagged                       Filter flagged requirements and print
                                  matches as tree/JSON in non-interactive
                                  workflows.
  --no-flag                       Filter unflagged requirements (Flagged:
                                  false or missing).
  --has-link                      Filter requirements that have one or more
                                  links.
  --no-link                       Filter requirements that have no links.
  --sub-domain TEXT               Filter by subsection name using case-
                                  insensitive prefix matching
                                  (repeatable/comma-separated).
  --targets-file TEXT             Path to a .txt/.conf/.md target list
                                  containing requirement IDs, domain tokens,
                                  and subsection tokens.
  --as-tree                       With --status: print a tree view only and
                                  exit instead of opening the interactive
                                  walk.
  --as-list                       With filters or explicit target tokens:
                                  print a flat list and exit.
  --table / --no-table            Print the summary table output (disable in
                                  automation with --no-table).
  --sort-profile [alpha-asc|standard|status-focus]
                                  Select a named interactive sort strategy
                                  catalog.  [default: standard]
  --totals                        Print aggregate status totals across all
                                  requirement files and exit.
  --totals-map TEXT               Custom rollup column equation, repeatable
                                  (for example --totals-map 'C1=I+V').
  --totals-config TEXT            Optional path to rollup config
                                  (.json/.yml/.yaml) containing rollup_map or
                                  rollup_equations.
  --as-json                       Print machine-readable JSON output for non-
                                  interactive workflows.
  --include-requirement-body / --no-requirement-body
                                  With --as-json --status: include full
                                  requirement body and line metadata (disable
                                  with --no-requirement-body).
  --resume-walk / --no-resume-walk
                                  Resume filtered interactive walkthrough
                                  position across runs.
  --strip-status-icons            One-time conversion: remove emoji prefixes
                                  from all status lines and regenerate
                                  summaries.
  --restore-status-icons          One-time conversion: restore canonical
                                  emoji-prefixed status lines and regenerate
                                  summaries.
  --seed-priorities               One-time migration: add default priority
                                  lines to requirements missing them.
  --seed-priority TEXT            Default priority used by --seed-priorities
                                  (for example p0, high, medium, low).
                                  [default: p3]
  --session-state-dir TEXT        Directory for persisted workflow state:
                                  system-temp, project-local, or a custom
                                  path.  [default: system-temp]
  --project-root DIRECTORY        Project root containing requirement
                                  documentation.  [default: .]
  --status-config TEXT            Path to a status catalog file (.yml, .yaml,
                                  or .json). Overrides .rqmd/statuses.yml
                                  auto-detection.
  --priorities-config TEXT        Path to a priority catalog file (.yml,
                                  .yaml, or .json). Overrides
                                  .rqmd/priorities.yml auto-detection.
  --docs-dir TEXT                 Directory (absolute or relative to
                                  --project-root) containing requirement
                                  markdown files. When omitted, rqmd auto-
                                  detects from the current working path.
  --id-namespace TEXT             Allowed header ID prefixes. Repeat or comma-
                                  separate values, for example --id-namespace
                                  R or --id-namespace AC,R.
  --verify-index                  Check that the requirements index
                                  (README.md) links match actual domain files;
                                  report stale links and orphan files.
  --bootstrap                     Initialize docs scaffold (index + starter
                                  domain file) and exit.
  --force-yes, --force-confirm    Auto-confirm scaffold initialization (non-
                                  interactive friendly).
  -h, --help                      Show this message and exit.
